Lesson 1 - HTML and Web Basics

Basics of the Web

The "web" or internet as it is known is a network of computers and other connected devices that are able to send requests to each other as well as web servers in an effort to communicate with each other.

A device sends a request out to a web server and the web server then uses what is known as DNS in an effort to connect that device to the web page. Once the device receives this communication from the webpage the browser displays that website on the screen.

HTML

HTML = Hypertext Markup Language. These are web documents on the internet. The web is consisted mostly of HTML documents. Websites are considered HTML documents. HTML is the language that computers use to interpret how to display the document to its user.

Tags and Elements

When you write HTML code you use things called tags and elements to format your web page. Without these tags and elements the web browser does not know how to interpret the commands that the developer wishes to display to users.

Inline vs. Block

Elements are either inline or block. Inline elements stay within the same line on the page while block elements have an invisibile box around them thus creating a format for the source.

Lesson 2 - Creating Structured Documents

Google Chrome Developer Tools

The web browser Google Chrome offers a tool Called Developer Tools. This allows the user to view the HTML code and CSS Style to inspect both languages and experiment with the code to make changes and learn how the HTML and CSS code was developed.

The Tree Structure

Remember that web pages are made up of "boxes." When these boxes are inside other boxes they are called nested elements. When elements are nested together they create a tree-like structure within the HTML. They have a releationship much like a mother or father has a relationship with their children.

HTML and Nesting

As you are reading the HTML code in a document you may notice that the coding itself has indentations. As the code gets more and more indented, this represents deeper nesting within that tree-like structure that we talked about with the elements of the HTML.

Text Editors

Programmers use programs called Text Editors to help them write their code. There are a number of text editors available on the web to use. Typically the text editing program makes the code easier to read with different colors and features that make coding for a programmer much easier than reading and writing a bunch of black text on a white background. Most editors give suggestions for HTML commands and elements as you type and also close your HTML tag for you when you write the opening tag so you do not forget as this is a common mistakke, especially for a beginning programmer.

Lesson 3 - Adding CSS Style and HTML Structure

Avoiding Repetition

There are a few of reasons to avoid repetition.

Consistency:

By making your HTML and web page consistent you give it good structure that allows you to avoid repetition. Using a CSS Sheet to develop your code allows you to avoid writing the same code over and over inside of your HTML to style the page. Instead of writing it multiple times you only have to write it once. The allows for more consistency throughout your document.

Avoid Errors:

Additionally, by reducing the amount of code you have to write by using a CSS sheet you give yourself the opportunity to reduce the amount of errors within your code.

Ease/Simplicity:

Simply put, when you avoid repetition it makes it easier to write and develop your code which allows you to focus more on the experience that you want to give your users.

CSS

CSS is also known as a Cascading Style Sheet. CSS gives the programmer the option to create the style of the page and elements. Elements can have class names and id's that are linked to the CSS stylesheet to keep the style of the page consistent and the coding simple.
